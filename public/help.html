MS OAuth2API 管理面板使用指南

🏠 首页 ⚙️ 管理面板 📖 API 文档

🚀 快速开始

1. 获取 OAuth2 凭据

在使用管理面板之前，您需要准备以下信息：

  • Refresh Token: 用于刷新访问令牌的长期凭据
  • Client ID: 您的应用程序客户端标识符
  • 邮箱地址: 要管理的 Outlook 邮箱地址
  • API 密码: 如果部署时设置了密码保护（可选）
💡 提示：这些凭据将安全地存储在您的浏览器本地存储中，不会发送到任何第三方服务器。

2. 配置连接

  1. 访问 管理面板
  2. 在"配置"标签页中填写您的 OAuth2 信息
  3. 点击"保存配置"按钮
  4. 点击"测试连接"验证配置是否正确

📧 功能说明

邮件管理

  • 获取最新邮件: 获取收件箱或垃圾箱中最新的一封邮件
  • 获取全部邮件: 获取指定文件夹中的所有邮件
  • 验证码识别: 自动识别并高亮显示邮件中的6位数字验证码
  • 完整邮件查看: 在新窗口中查看邮件的完整 HTML 内容

发送邮件

  • 多收件人支持: 支持向多个邮箱地址发送邮件（用逗号分隔）
  • HTML 格式: 支持发送纯文本或 HTML 格式的邮件
  • 实时状态: 显示发送进度和结果反馈

邮箱清理

  • 清空收件箱: 批量删除收件箱中的所有邮件
  • 清空垃圾箱: 批量删除垃圾箱中的所有邮件
  • 安全确认: 执行清理操作前会要求确认
⚠️ 警告：邮箱清理操作是不可逆的，请在执行前确保您真的需要删除这些邮件。

🔧 技术说明

支持的邮箱类型

  • Microsoft Outlook
  • Hotmail
  • Live.com
  • 其他支持 Microsoft Graph API 的邮箱服务

API 接口

管理面板调用以下 API 接口：

GET /api/mail-new - 获取最新邮件 GET /api/mail-all - 获取全部邮件 POST /api/send-mail - 发送邮件 GET /api/process-inbox - 清空收件箱 GET /api/process-junk - 清空垃圾箱

数据安全

  • 所有配置信息仅存储在浏览器本地
  • 支持 HTTPS 加密传输
  • 不会记录或存储您的邮件内容
  • 可随时清除本地配置信息

❓ 常见问题

Q: 为什么测试连接失败？

A: 请检查以下几点：

  • 确保 Refresh Token 和 Client ID 正确
  • 确保邮箱地址格式正确
  • 检查网络连接是否正常
  • 确认 API 服务是否正常运行

Q: 如何获取 OAuth2 凭据？

A: 您需要在 Microsoft Azure 门户中注册应用程序并配置相应的权限。详细步骤请参考 Microsoft 官方文档。

Q: 支持哪些浏览器？

A: 支持所有现代浏览器，包括 Chrome、Firefox、Safari、Edge 等。建议使用最新版本以获得最佳体验。

Q: 数据会被保存在服务器上吗？

A: 不会。所有配置信息都存储在您的浏览器本地存储中，服务器不会保存任何用户数据。
